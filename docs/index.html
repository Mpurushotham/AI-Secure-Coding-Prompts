Why it matters

🤔 Why · ⏰ When · 🎯 How it helps

AI assistants generate plausible code, not secure code. They optimize for "does it run?", quietly skip the controls that don't show up in a demo, and will confidently invent APIs and packages that don't exist. Reviewing every generated line doesn't scale — so this library moves security into the model's first draft.

💡

Why this exists

Left unconstrained, assistants ship SQL injection, hardcoded secrets, missing authorization, and disabled TLS checks. Each prompt gives the model a concrete, framework-native rule set before it writes anything.

⏰

When to use it

Paste the matching prompt before you ask an assistant to write, refactor, or review code — as a system prompt or rules file (CLAUDE.md, .cursorrules). Several prompts also work at design and review time.

🎯

How it helps

Concrete beats comprehensive. Real APIs, real config keys, and named anti-patterns — not vague "best practices" — plus a self-check the model runs before returning code. Fewer vulns, less review back-and-forth, a consistent bar.

Workflow

📋 How to use it — four steps

Find your stack

Open the category and pick the prompt for your framework, cloud, or concern.

Paste the prompt block first

Drop the fenced text block into your assistant as a system prompt or rules file — before you ask for code.

Give it your real task

"Build a file-upload endpoint", "review this PR", "write the Terraform for our VPC." The rules stay active all session.

Stack prompts for cross-domain work

e.g. python.md + sql-injection-prevention.md + jwt-security.md for a Flask API with auth.

The contract

🧱 Every prompt follows the same shape

Predictable structure means predictable behavior — the model knows exactly what "secure" means for the task.

  • 🎭 Role & scope

    Who the AI acts as and which code the rules apply to.

  • 📐 Hard requirements

    Numbered, specific, framework-native controls — real APIs and config keys, not platitudes.

  • 🚫 Forbidden patterns

    Things the AI must never emit, even if asked casually.

  • ✅ Verification

    A self-check the AI runs before returning code.

  • 🪝 Escape hatch

    If a requirement can't be met, the AI must say so — instead of silently degrading security.
